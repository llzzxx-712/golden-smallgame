## 第一阶段：需求 → 设计

### 初始需求
```
有一个棋盘，玩家要从起点移动到金矿，挖了金矿再回来。
每个组会发一些游戏币来购买路上需要的各种物资和道具，
运气不好就会渴死饿死在沙漠里。

画面可以参考小型独立游戏的卡通风，
在保证还可以的画面和游戏性的情况下使用下载量相对较小的方案。
```

## 核心设计与算法

### 地图随机生成
- 8列 × 5行网格，营地固定左端，金矿固定右端
- 每列随机 1~5 个节点，相邻列最近行优先连接 + 随机对角边
- BFS 验证连通性，不连通则回退补充路径
- 难度影响绿洲/沙暴生成概率

### 迷雾系统
- 初始仅显示相邻节点（距离1），隐藏节点显示为 `?`
- 移动后逐步揭示，指南针揭示距离2
- 事件"捷径"揭示距离3

### 分数累加消耗制
- 基础消耗速率 1.0/步，游牧民 0.8/步
- 内部维护小数累加器，累加满 1 才扣实物
- 5步消耗4 = 精确 20% 减少

### 事件概率公式
```
好事件权重 = 0.40 + 难度修正 + 节点修正
坏事件权重 = 0.35 - 难度修正 - 节点修正
中立权重 = 1 - 好 - 坏
```

## 给开发者的提示

如果你想继续修改这个游戏：

- **数值调整：** 编辑 `js/game.js` 中的 `DIFFICULTIES`（难度参数）、`ITEM_TEMPLATES`（道具价格/效果）、`CHARACTERS`（角色属性）
- **事件修改：** 编辑 `js/events.js` 中的 `EVENTS` 对象
- **成就添加：** 编辑 `js/achievements.js`，在 `ACHIEVEMENTS` 数组中添加并写检查逻辑
- **地图规模：** 编辑 `js/map.js` 中 `generateMap` 的 `cols`/`rows`
- **本地运行：** 双击 `index.html` 或 `python3 -m http.server 8080`
